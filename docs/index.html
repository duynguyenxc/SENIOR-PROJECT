Veg Buffet — User Manual

A complete guide to installing, using, and maintaining the Veg Buffet vegetarian restaurant ordering system. Built as a senior capstone project.

Get Started → View Live Demo ↗

What Is Veg Buffet?

Veg Buffet is a web-based takeout ordering system for a vegetarian restaurant. Customers browse a rotating weekly menu, place and pay for takeout orders online, then track their order status in real time. On the restaurant side, staff and administrators use a separate internal portal to process orders, manage the menu, and review sales analytics.

The application is live at: https://veg-buffet-web.onrender.com/

Source code is hosted on GitHub. Clone or download it from: https://github.com/duynguyenxc/SENIOR-PROJECT

ℹ First Load Note
The live site runs on Render's free tier. After a period of inactivity, the first request may take up to 60 seconds while the server wakes up. Subsequent requests are fast.

What the System Does

The application is split into two sides:

Customer-Facing Side (public website)

  • Browse the weekly buffet menu organized by day
  • View available takeout sets with prices and descriptions
  • Create an account and log in
  • Add items to a cart and complete checkout with a chosen pickup time
  • Pay securely via Stripe Checkout (sandbox mode)
  • Receive order confirmation and track the order status in real time
  • View full order history from the account area

Internal Staff Portal (/admin/)

  • Staff — view the live order queue, update order statuses, review history
  • Super Admin — all staff capabilities plus: manage the weekly menu, manage takeout sets, create/disable staff accounts, view sales analytics

Payment Integration — Stripe Checkout

One of the most significant technical achievements in this project is the integration of Stripe Checkout — a production-grade, industry-standard payment gateway used by companies like Amazon, Shopify, and Lyft. This makes Veg Buffet function as a real e-commerce application, not just a prototype.

What Stripe Checkout Is

Stripe Checkout is a hosted, pre-built payment page provided by Stripe. When a customer clicks "Proceed to Payment," the application creates a secure payment session on Stripe's servers (via an API call from PHP), then redirects the customer to Stripe's own payment page hosted at checkout.stripe.com. The customer enters their card details directly on Stripe's page — not on the Veg Buffet server. After the payment succeeds or is cancelled, Stripe redirects the customer back to the application.

Why It Is Secure

This architecture is intentionally designed to keep sensitive financial data off our server entirely:

  • No card data touches the application server. Card numbers, CVCs, and billing details are entered on Stripe's hosted page and processed by Stripe's infrastructure. The Veg Buffet server never sees, stores, or transmits card information.
  • PCI DSS compliance is handled by Stripe. The Payment Card Industry Data Security Standard (PCI DSS) is a strict set of security requirements for handling card data. By offloading payment to Stripe's hosted page, the application is considered "SAQ-A" compliant — the lightest possible compliance burden — because it never touches raw card data.
  • HTTPS enforced by Stripe. Stripe's checkout page is always served over HTTPS with TLS encryption. No insecure transmission of payment data is possible.
  • Payment verification is server-side. After the customer is redirected back, the application verifies the payment status by calling the Stripe API from the server using the secret key — not by trusting any URL parameter or client-side data. This prevents anyone from faking a successful payment.

Why Stripe Was Chosen

  • Sandbox mode for safe testing. Stripe provides a fully functional test environment with dedicated test API keys and test card numbers. The entire payment flow — from checkout to confirmation — can be tested without any real money, no real bank account, and no approval process.
  • No server-side card handling required. Stripe Checkout handles the payment form, validation, and processing on its own servers. This significantly reduces the complexity and security risk of the implementation.
  • No third-party SDK required. The entire payment integration is implemented in a single file (web/lib/payment.php) using PHP's built-in cURL extension to call Stripe's REST API directly. No Composer, no package manager, and no external library files are needed.
  • Industry standard. Stripe is one of the most widely used payment processors in the world. Integrating it in a capstone project demonstrates real-world full-stack development skills.
