User Guide

This section describes the user interface structure and functionality for all three types of users: Customer, Staff, and Super Admin. Each role has different needs and different pages available to them.

Customer Guide

Customers are the general public who visit the website to place takeout orders. A customer account is required to place an order. Account creation is free and instant.

Step 1 — Create an Account

Click Create account in the navigation bar. Fill in the form and click Create Account.

  • Email — used as the login username; must be unique
  • Full name — displayed on the order for staff
  • Phone — optional; for staff to contact you about the order
  • Password — minimum 6 characters; stored as a bcrypt hash in the database
ℹ Already have an account?
Click Login in the navigation bar. Enter your email and password to sign in.

Step 2 — Browse the Weekly Menu

Click Menu in the navigation bar to see what the restaurant is serving this week. The menu is organized by day — click any day tab to see that day's dishes. Clicking a dish name expands it to show the full ingredient list.

ℹ Menu vs. Takeout
The Menu page is for information only — you cannot order directly from it. Ordering happens through the Takeout page. Use the menu to decide which dishes to include in a Custom Takeout Box.

Step 3 — Choose a Takeout Set

Click Takeout in the navigation bar to see available sets. There are two types:

TypeDescriptionExamples
Fixed sets Pre-packaged combinations at a set price. Contents are decided by the restaurant. Vegan Delight Box ($14.99), Spicy Thai Combo ($15.00), Gluten-Free Harvest ($16.50)
Custom Takeout Box You list up to 6 dishes from the weekly menu. Staff fulfill the first 6 items entered. Custom Takeout Box ($18.00)

For each set, you can optionally enter a special note (e.g., "no peanuts", "less spicy", "pack sauce separately") before clicking Add to Cart.

For the Custom Takeout Box, type your chosen dish names in the text area (one per line or comma-separated). Staff will prepare the first 6 dishes listed.

⚠ You must be logged in to add items
If you are not logged in, clicking Add to Cart redirects you to the login page. Log in first, then return to the Takeout page.

Step 4 — Review Your Cart

Click Cart in the navigation bar. On this page you can:

  • Review all items, quantities, and prices
  • See the total amount
  • Enter your name, phone number, and preferred pickup time
  • Add any special instructions or allergy notes for the entire order
  • Click Proceed to Payment to go to Stripe checkout

Step 5 — Pay via Stripe

Clicking Proceed to Payment takes you to Stripe's hosted payment page. Enter card details and complete the payment. The system is in test mode — no real money is charged.

Card FieldTest Value

ExpiryAny future date

After payment, Stripe redirects back to the order confirmation page showing your order number and a real-time status tracker.

Step 6 — Track Your Order

The confirmation page shows the current order status and updates automatically without needing to refresh the page.

Paid → Preparing → Ready for Pickup → Completed

You can also view all past orders at any time by clicking My Orders in the navigation bar (only visible when logged in).

Staff Guide

Staff members process incoming orders through the internal portal. Staff accounts are created exclusively by the Super Admin — staff cannot self-register.

Staff have access to: Order Queue (Dashboard) and Recent Orders (History). They do not have access to menu management, reports, or staff account management.

Logging In

Go to the staff portal login page:

  • Live: https://veg-buffet-web.onrender.com/admin/login.php
  • Local: http://localhost:8080/admin/login.php

Enter your staff email and password, then click Access Dashboard.

The Order Fulfillment Dashboard

After logging in, the dashboard shows:

  • Today's Revenue — total from paid orders today
  • Completed Today — how many orders have been completed
  • Active Queue — number of orders in progress
  • Live order cards — all active orders sorted by pickup time (earliest first)

Reading an Order Card

Each order card shows:

  • Order number and customer name (e.g., "Order #5 — John Smith")
  • Customer phone number
  • Total amount
  • Pickup time (prominently on the right)
  • Current status (color-coded badge)
  • All items in the order with quantities and any item-level notes
  • Special instructions and allergy notes (shown in a highlighted callout if present)

Processing an Order (Advancing Status)

Staff advance each order through three stages by clicking the button that appears on the card:

Current StatusButton to ClickNew StatusWhen to Click
Paid Start Preparing Preparing When you begin preparing this order in the kitchen
Preparing Mark Ready Ready When the order is fully packed and ready for the customer
Ready Complete Order Completed When the customer has picked up the order

Once marked Completed, the order disappears from the queue and moves to history. The customer's status tracker also updates in real time.

Order History

Click Recent Orders in the navigation bar to view all past orders (newest first). Click any order to see its full details, including the Stripe payment reference ID.

Super Admin Guide

The Super Admin has full access to everything the Staff can do, plus four additional management areas:

  1. Menu Builder — manage the weekly dish menu by day
  2. Takeout Sets — manage the takeout product catalog
  3. Manage Staff — create, disable, and reset passwords for staff accounts
  4. Reports — view sales analytics and business insights

Login credentials (default, change after setup):

  • Email: admin@vegbuffet.com
  • Password: password

Menu Builder

Go to Menu Builder. This page controls which dishes appear on each day of the weekly menu.

How to use:

  1. Click a day name to select it. The current dishes for that day appear on the right.
  2. To remove a dish from the day, click Remove next to it.
  3. To add a dish, fill in the Dish Name, Description, and optionally upload an image, then click Add.
ℹ Dishes are shared across days
The same dish can appear on multiple days. If the dish name already exists in the database, the system reuses the existing record and simply links it to the selected day.

Takeout Sets Management

Go to Takeout Sets to manage the product catalog customers see on the Takeout page.

  • Add a set: fill in name, description, price, image, and availability, then click Add Set.
  • Toggle availability: each set has an Available/Unavailable toggle. Unavailable sets are hidden from customers.
  • Edit a set: click Edit to modify any field.
  • Delete a set: click Delete to permanently remove a set.

Staff Account Management

Go to Manage Staff to manage who has access to the internal portal.

Creating a Staff Account

  1. Scroll to the Create Staff Account section.
  2. Enter the staff member's email and a temporary initial password.
  3. Click Create Staff. The account is immediately active.
  4. Share the credentials with the new staff member securely.

Disabling an Account

Click Disable next to an account. The staff member can no longer log in. Their account is not deleted — click Enable to restore access.

Resetting a Password

Click Reset Password next to an account. Enter and confirm a new temporary password. Share it with the staff member so they can log in.

Cancelling an Order

On the Order Queue dashboard, Super Admins see a Cancel Order button on each order card. This immediately sets the order to Cancelled status.

⛔ Cancelling does not issue a Stripe refund
Cancelling an order only changes its status in the database. To refund the customer, log into the Stripe Dashboard and issue a refund manually for that payment.

Accessibility

The Veg Buffet web interface includes the following accessibility considerations:

  • Semantic HTML — uses <nav>, <main>, <section>, <button>, <label>, and other semantic elements that provide baseline screen reader compatibility.
  • Labeled form inputs — every form field has an associated <label> element, which is required for screen reader users to understand what each input expects.
  • Text alongside color — status indicators (Paid, Preparing, Ready, Completed, Cancelled) always include the status name as text, not just color. Color is not the sole means of conveying information.
  • Keyboard navigability — all interactive elements (buttons, links, form fields) are reachable by keyboard using the Tab key, consistent with standard browser behavior.
  • Responsive layout — the layout adapts to mobile screens, tablets, and desktops. No specialized device or screen size is required.
  • No specialized hardware required — the application runs entirely in a standard web browser and requires no specialized hardware.

The interface is self-explanatory by design, with informative button labels (e.g., "Start Preparing", "Mark Ready", "Complete Order") and descriptive error messages shown inline when form validation fails.
